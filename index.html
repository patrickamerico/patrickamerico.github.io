Painel Hospitalar COVID-19 | Dashboard - Weknow

🏥 O Desafio

Hospitais precisavam acompanhar de forma precisa e em tempo real os indicadores da COVID-19, enfrentando desafios como:

  • Dados fragmentados no ERP Hospitalar Tasy
  • Necessidade de atualização diária automática
  • Falta de visualização segmentada por faixa etária, sexo, cidade ou tipo de atendimento
  • Tomada de decisão lenta por ausência de dashboards consolidados

      • O objetivo era criar um painel interativo que centralizasse informações críticas, permitindo decisões rápidas e baseadas em dados.

⚙️ Minha Contribuição Estratégica

Como Tech Lead do projeto, liderei:

  • Arquitetura de uma solução de BI corporativo integrando dados do ERP Tasy com fontes complementares em uma modelagem multidimensional
  • Definição de governança de dados e pipelines de ETL para garantir acurácia e rastreabilidade
  • Implementação de alertas automáticos para situações críticas
  • Otimização de performance para consultas em tempo real com grandes volumes históricos
  • Mentoria do time na adoção de boas práticas de engenharia de dados

📊 O Resultado

  • Redução de 30% no tempo médio de geração de relatórios estratégicos
  • Aumento para 99,9% de confiabilidade dos dados para tomada de decisão
  • Monitoramento em tempo real de casos suspeitos, confirmados, óbitos e ocupação de leitos
  • Capacidade de resposta rápida a situações críticas com alertas automatizados

Arquitetura de Pipeline de Dados — Engenharia, Governança e Qualidade

📊 O Desafio

A organização operava com dados de produção distribuídos em arquivos CSV heterogêneos, sem padronização semântica, validação estrutural ou controle de qualidade.

    Esse cenário gerava:
  • Inconsistência em indicadores
  • Risco de decisões baseadas em dados incorretos
  • Ausência de rastreabilidade
  • Alto custo operacional na preparação manual dos dados
  • Processo manual e não reprodutível

O desafio não era apenas processar dados — era estabelecer uma base confiável de engenharia de dados com governança e evolução estruturada.

⚙️ Minha Contribuição Estratégica

  • Documentação de processos
  • Estruturação de pipeline modular seguindo princípios de separação de responsabilidades (extração, transformação, persistência)
  • Modelagem relacional otimizada para consultas analíticas
  • Definição clara de tipagem e integridade de dados no banco SQLite
  • Versionamento incremental da solução (v1 → v5)

📊 O Resultado

  • Redução de semanas para algumas horas no tempo de preparação de dados para análise
  • Datasets confiáveis e atualizados diariamente para dashboards estratégicos
  • Automação completa eliminando intervenções manuais propensas a erros
  • Base preparada para evolução com Apache Airflow ou migração para PostgreSQL

Previsão de Churn em Telecom

📉 O Desafio

Uma empresa de telecom enfrentava altas taxas de cancelamento mensal, mas as ações de retenção eram reativas e genéricas. Era necessário um modelo preditivo para identificar clientes com alto risco de churn e permitir ações proativas e personalizadas.

⚙️ Minha Contribuição Estratégica

  • Conduzi Análise Exploratória de Dados (EDA) para identificar padrões comportamentais
  • Realizei pré-processamento de variáveis categóricas para adequação ao modelo
  • Estruturei pipeline de modelagem supervisionada para classificação
  • Treinei modelo baseado em Scikit-learn utilizando algoritmo Random Forest
  • Avaliei performance com métricas de classificação (Acurácia, Precisão, Recall, F1-Score)
  • Analisei matriz de confusão para entendimento de erros (FP/FN)
  • Extraí e interpretei feature importance para gerar insights estratégicos
  • Organizei o projeto com separação entre notebook exploratório e script de produção
  • Estruturei ambiente com dependências versionadas

📊 O Resultado

  • Aumento de pelo menos 10% na retenção de clientes identificados como alto risco
  • Campanhas de marketing direcionadas
  • Modelo interpretável permitindo entender os principais drivers de cancelamento
  • Pipeline automatizado para re-treinamento periódico do modelo

Validação de CPF Integrado ao ERP

🔍 O Desafio

No processo de cadastro de pacientes em ERP de saúde, a entrada manual de CPFs gerava:

  • Erros de digitação e inconsistências cadastrais
  • Risco de dados incorretos ou fraudes

O desafio era criar uma solução que unisse automação, segurança e agilidade, garantindo confiabilidade dos dados cadastrais e integridade do sistema.

⚙️ Minha Contribuição Estratégica

  • Liderança no desenvolvimento de microsserviço dedicado para validação de CPF
